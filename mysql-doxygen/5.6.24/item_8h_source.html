1495  german character for double s is equal to 2 s.
1496 
1497  The default is that an item is not allowed
1498  in a partition function. Allowed functions
1499  can never depend on server version, they cannot depend on anything
1500  related to the environment. They can also only depend on a set of
1501  fields in the table itself. They cannot depend on other tables and
1502  cannot contain any queries and cannot contain udf's or similar.
1503  If a new Item class is defined and it inherits from a class that is
1504  allowed in a partition function then it is very important to consider
1505  whether this should be inherited to the new class. If not the function
1506  below should be defined in the new Item class.
1507 
1508  The general behaviour is that most integer functions are allowed.
1509  If the partition function contains any multi-byte collations then
1510  the function check_part_func_fields will report an error on the
1511  partition function independent of what functions are used. So the
1512  only character sets allowed are single character collation and
1513  even for those only a limited set of functions are allowed. The
1514  problem with multi-byte collations is that almost every string
1515  function has the ability to change things such that two strings
1516  that are equal will not be equal after manipulated by a string
1517  function. E.g. two strings one contains a double s, there is a
1518  special german character that is equal to two s. Now assume a
1519  string function removes one character at this place, then in
1520  one the double s will be removed and in the other there will
1521  still be one s remaining and the strings are no longer equal
1522  and thus the partition function will not sort equal strings into
1523  the same partitions.
1524

1526  check that the field types are valid, next check that the partition
1527  function is valid. The current set of partition functions valid
