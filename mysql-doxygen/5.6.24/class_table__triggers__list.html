◆ m_has_unparseable_trigger

bool Table_triggers_list::m_has_unparseable_trigger
private

This flag indicates that one of the triggers was not parsed successfully, and as a precaution the object has entered a state where all trigger access results in errors until all such triggers are dropped. It is not safe to add triggers since we don't know if the broken trigger has the same name or event type. Nor is it safe to invoke any trigger for the aforementioned reasons. The only safe operations are drop_trigger and drop_all_triggers.
